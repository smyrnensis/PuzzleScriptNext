3D

3D mode extends ordinary PuzzleScript with one extra spatial axis. Rules, objects, legends, collision layers, win conditions, commands, tags, mappings, and level flow keep their normal PuzzleScript meanings. The new pieces are the three_dimensions prelude flag, 3D directions, 3D level slices, 3D sprites, and camera metadata.

Starting a 3D Game

Add three_dimensions in the prelude. This enables the 3D parser, compiler, runtime, and renderer path. A first 3D example should show a real 3D asset and a real 3D level, so this starter uses 5-by-5-by-5 sprites and five level slices.

The object sprites above are written as five 5-by-5 slices separated by semicolon lines. The level is written the same way: each slice is a familiar 2D grid, and the semicolon lines stack those grids into a 3D board.

Directions

3D mode keeps the 2D directions up, down, left, right, and action, then adds front and back. The 2D movement bits keep their original order; front and back are appended as the extra spatial directions.

front [ player | no wall ] -> [ | player ]
back  [ player | no wall ] -> [ | player ]

The 3D shorthand horizontal means left, right, front, and back. The shorthand orthogonal means all six spatial directions.

Relative Directions

Relative rule markers still work, but 3D has two more relative markers. The standard frame is:

  • > means right.
  • < means left.
  • ^ means front.
  • v means back.
  • o means up.
  • x means down.
[ > player | crate ] -> [ > player | > crate ]
[ o player | crate ] -> [ o player | o crate ]

As in 2D, the compiler expands relative rules into concrete directions. In 3D, that expansion uses 3D oriented rule frames rather than 2D rotations.

3D Levels

3D Sprites

Objects use 3D sprite data by writing several ordinary 5-by-5 sprite slices separated by semicolon lines. Each slice uses the same color index rules as a 2D sprite: digits select colors and . is transparent. Five slices make a 5-by-5-by-5 sprite.

If an object only has a normal 2D sprite, the renderer uses that as its visible sprite data. Use 3D sprite slices when the shape itself needs depth.

Camera Metadata

3D camera settings are prelude metadata.

perspective_camera
camera_angle 35 60
camera_zoom 1.25
camera_view_angle 40

camera_angle takes yaw and pitch in degrees. camera_zoom changes framing. camera_view_angle controls the perspective camera view angle in degrees. Use orthographic_camera if you do not want perspective size distortion.
